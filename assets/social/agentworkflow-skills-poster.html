AI 研发流程技能

AgentWorkflow Skills

让 AI 研发助手真正按工程流程交付：先确认需求，再拆计划，再写代码，最后验证、留痕、交接。

完整交付路径Reference → DSL → Plan → AT-T → Verify → Handoff / Git
01
参考资料归档
收集产品文档、工程文件、接口材料，形成统一参考入口。
02
DSL 多文件建模
覆盖需求、页面、交互、事件、边界、验收标准。
03
工程师确认
DSL 未确认不进入研发计划，减少方向偏差。
04
前后端 Plan
按前台前端、后台前端、后端服务拆出可执行任务。
05
AT-T 原子任务
每个任务开始前先问清需求，确认后再写代码。
06
验证与交接
lint / test / build / e2e，记录 bug、变更、版本和 handoff。
✓需求先审

需求、口述新增、研发变更统一进入需求表，并回写 DSL、Plan、任务清单。

?任务先问

开始每个子任务前像真实研发一样追问边界，工程师确认后才动代码。

↔前后端同步

同步中心共享 DSL、Plan、接口契约、Bug、Handoff 和任务看板。

#工程师索引

ENGINEERING_INDEX + FILE_INDEX 记录项目全部代码文件用途，方便人工接手。

!Bug 留痕

所有问题进入 AI_BUG_LOG，包含复现、影响、修复、验证和回归建议。

◎记忆交接

上下文压缩、项目记忆、聊天记忆和 Handoff 串联，换会话也能连续研发。

多 Agent 协同机制 Sync Center First
前台前端 Agent 认领用户端页面、状态、接口调用、体验验收任务。
后台前端 Agent 认领管理端 CRUD、权限视图、运营流程、表单校验。
后端 Agent 认领服务模型、API 契约、数据库、安全和集成测试。
