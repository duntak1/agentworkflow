# Dependency Review（依赖准入）

用途：新增或升级依赖前，记录许可证、安全、维护状态、替代方案和采用理由。

## 维护规则

- 能用项目已有工具、官方 SDK、框架能力解决时，优先不加新依赖。
- 新依赖必须写明 license、安全风险、维护活跃度、替代方案和确认人。
- 高风险依赖必须关联 Policy Decision。
